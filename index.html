PROJECT_01

Mini Warman Robot

Collaborated in a team of five to design and build a small-scale automated transport system using MDF and electronic components such as an Arduino Uno, DC motors, and servo motors, with the objective of collecting, navigating, and depositing two tennis balls into a target hole. Our forklift inspired design used 3D printed forks that were attached to gears and a DC motor to pickup and drop off the ball. I contributed significantly in a manufacturing role by assisting with CAD design, generating laser-cutting files, and fabricating components for system assembly. Overall, this project was a valuable first hands-on experience with combining controllors, sensors and actuators to create a automated system. Additionally, it taught me valuable lessons regarding the importance of teamwork to yield results and reflecting on areas for further improvement.

PROJECT_02

Automated Irrigation System

Built an automated closed-loop irrigation system that reads soil moisture and delivers accurate volumes of water to a plant when dry soil is detected. Operating on an Arduino Uno, the system uses precise, interrupt-driven volumetric control using a Hall-Effect water flow sensor along with a capacitive soil moisture sensor for real-time monitoring. The control logic was programmed using C++, and uses a finite state machine design that contains 3 states: IDLE, OPERATING, FAULT. The system operates in IDLE state when the soil moisture is below the set soil moisture threshold, the OPERATING state when it is above the threshold, and the FAULT state when no water is flowing through the tubing or an error has been flagged. Working on this personal project let me experience the struggles of debugging along with the satisfaction of applying the theory I learn into a real-life control system.

PROJECT_03

Vex Robot Control

Programmed an autonomous VEX robot with a partner using C++ to simulate a warehouse forklift tasked with transporting a payload. The system integrated closed-loop PI/P control loops for straight-line driving, precise turning, and arm manipulation alongside ultrasonic sensor feedback for slip-resistant movement and a 3-light-sensor array for path following. During our program demonstration, individual sub-functions succeeded in isolation, but due to some unoptimised gains and imperfect programming, the full task was not completed as expected. However, these shortcomings provided valuable troubleshooting experience, highlighting the importance of early system testing and the need to upgrade basic threshold logic to continuous PID loops for robust navigation.

PROJECT_04

ADC Conversion Project

Worked with a partner to develop real-time software that quantifies ADC offset error across varying prescaler settings, enabling analysis of the trade-off between sampling speed and conversion accuracy. Using an ATmega328P (Arduino Uno), we wrote low-level C code to interface directly with hardware registers, bypassing standard libraries for precise control of the sampling process. We implemented a finite-state machine driven by tactile inputs to switch between recording analog signals into a ring buffer and transmitting data to a computer. The collected samples were then processed in MATLAB to evaluate and visualise the relationship between clock speed and offset error. This project strengthened my experience in real-time embedded systems, efficient memory management, and understanding the practical limitations of microcontroller hardware.
